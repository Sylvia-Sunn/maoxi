# 开发文档 — 毛溪个人网站

> 本文档记录本项目（`site/`）的技术实现，供后续维护 / 更新内容时参考。
> 设计来源：`../project/毛溪个人网站-standalone.html`（Claude Design 导出的原型），
> 需求沟通记录见 `../chats/chat1.md`。

## 1. 项目概述

毛溪老师的个人学术主页，单页应用，中英双语（右上角"中 / EN"切换），共 7 个锚点分区：

| 区块 | 锚点 | 内容 |
|---|---|---|
| 简介 | `#about` | 头像、姓名、职称、任职、邮箱、简介 |
| 近况 | `#news` | 可滚动的近况时间线（6 条） |
| 研究 | `#research` | 三大研究方向卡片 + 主要科研项目 |
| 论著 | `#publications` | 精选论文（含 DOI 链接）、专著/教材、专栏、发明专利 |
| 创业 | `#ventures` | 五项创业孵化时间线 |
| 履历 | `#cv` | 教育与工作经历时间轴 + 学术/社会任职 |
| 联系 | `#contact` | 邮箱、工作单位（外链） |

视觉方向参照 [luost26/academic-homepage](https://github.com/luost26/academic-homepage)：白底单栏（880px 居中）、无衬线字体、单一蓝色强调色 `#2b5f8c`、圆角卡片、细分隔线。

## 2. 技术栈

- **React 19** + **Vite 8**（`@vitejs/plugin-react`）
- 纯 CSS（无 CSS 框架 / CSS-in-JS），单一样式表 `src/App.css`
- **oxlint** 做代码检查
- 无路由库 — 单页 + 锚点导航（`<a href="#section">` + `scroll-margin-top`）
- 无状态管理库 — 语言状态是 `App.jsx` 里的一个 `useState`

## 4. 本地开发

```bash
cd site
npm install       # 安装依赖
npm run dev       # 启动开发服务器（默认 http://localhost:5173）
npm run build     # 生产构建，输出到 dist/
npm run preview   # 本地预览构建产物
npm run lint       # oxlint 静态检查
```

## 5. 更新网站内容

**所有文案都集中在 `src/data/content.js`**，不需要碰组件代码。每一项内容都用
`t('中文', 'English')` 的写法同时提供中英文，例如：

```js
const news = [
  { date: '2026', text: t('论文《民国广告符号的视觉现代性》被期刊 Design History 接收。',
                           'Paper on the visual modernity of Republican-era advertising symbols accepted at Design History.') },
  // 在数组开头插入新条目即可新增一条"近况"
];
```

按区块对应的数组 / 对象：

- `nav` — 顶部导航文案（一般不需要改）
- `headerRoles` — 简介区姓名下方的身份行（当前 3 条）
- `news` — 近况列表（时间倒序，插入新条目放最前面）
- `areas` — 研究方向卡片（当前 3 个，编号 `no` 需手动保持 01/02/03）
- `projects` — 主要科研项目
- `papers` — 精选论文；`teaserBg` 是缩略图占位的渐变色（CSS `linear-gradient`），
  如果之后有论文配图，把 `Publications.jsx` 里 `paper-teaser` 的
  `style={{ background: pp.teaserBg }}` 换成一张 `<img>` 即可
- `books` / `patents` — 专著教材 / 发明专利
- `ventures` — 创业孵化时间线
- `timeline` — 履历时间轴（教育与工作经历）
- `appointments` — 学术与社会任职
- `t` — 零散的界面文案（各区块标题、按钮文字等）

改完保存，`npm run dev` 会自动热更新，两种语言都要看一遍确认没有漏翻译。

## 6. 样式与设计规范

主要设计变量（写死在 `App.css` 里，未抽成 CSS 变量，因为只有一处调色）：

- 强调色：`#2b5f8c`（用于链接、日期、编号、边框高亮等）
- 正文色：`#2f3a4a` / 标题色：`#16233a` / `#1b2739`
- 分隔线：`#eceff3` / `#eef1f4`
- 内容最大宽度：`880px`，两侧 `padding: 0 24px`
- 圆角卡片：`border-radius: 14px`；胶囊按钮：`border-radius: 999px`

原型中用了内联样式的 `style-hover` 伪属性（Claude Design 私有语法），本实现改写成
了真正的 CSS `:hover` 规则（见 `App.css` 中 `.nav-link:hover`、`.pill:hover`、
`.area-card:hover`、`.contact-card:hover` 等）。

字体通过 Google Fonts 正常引入（`index.html` 的 `<head>`），不再像原型 standalone
导出那样把字体 / 图片转成 base64 内联——头像是真实文件
（`src/assets/mao-xi-portrait.png`），构建时由 Vite 正常处理、加哈希文件名。

## 7. 部署

`npm run build` 产出的 `dist/` 是纯静态文件，可以直接部署到任意静态托管：
Vercel / Netlify / GitHub Pages / Nginx 等，无需服务端渲染或数据库。

如果部署到子路径（不是域名根目录），需要在 `vite.config.js` 里设置 `base`。
